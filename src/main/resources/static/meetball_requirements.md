# 밋볼 (MeetBall) MVP 기능 요구사항 명세서

## 1. 프로젝트 개요

- **서비스명**: 밋볼 (가제)
- **서비스 한 줄 소개**: 프로젝트를 준비하는 사용자가 자신의 역할과 기술에 맞는 프로젝트를 찾거나 직접 팀원을 모집할 수 있는 프로젝트 기반 팀원 모집 · 매칭 플랫폼
- **문서 목적**:
  - 페이지별 핵심 기능 명확히 정의
  - 사용자별 이용 가능한 기능과 제한사항 정리
  - MVP 범위와 후순위 기능 구분
  - 와이어프레임 및 개발 작업의 기준 마련

---

## 2. 사용자 구분 및 권한

| 구분 | 주요 권한 및 이용 가능 기능 |
| --- | --- |
| **비회원** | 프로젝트 목록 및 상세 조회 <br>*(관심 등록, 지원, 등록, AI 추천은 로그인 필요)* |
| **회원(팀원/팀장 공통)** | 프로필 등록/수정, 프로젝트 탐색, 관심 등록, 프로젝트 지원, AI 추천 프로젝트 조회, 리뷰 작성 및 별점 확인 |
| **회원(팀장)** | 프로젝트 등록/수정/삭제, 지원자 조회/승인/거절, 첨부파일 업로드/관리, 프로젝트 운영/관리, 종료 후 리뷰 확인 |

---

## 3. 공통 요구사항

- **[0] FR-COM-01. 소셜 로그인**: 구글 계정을 통한 로그인 및 회원가입 (타 플랫폼은 후순위).
- **[0] FR-COM-02. 로그인 유도**: 비회원의 회원 전용 기능 접근 시 로그인 페이지/모달 이동.
- **[0] FR-COM-03. 로그인 상태 유지**: 새로고침 이후 상태 유지.
- **[0] FR-COM-04. 로그아웃**: 로그인 사용자의 로그아웃.
- **[✅] FR-COM-05. 권한 구분**: 팀원/팀장에 따른 기능 접근 제어, 비로그인 유저 접근 제한.
- **[0] FR-COM-06. 프로필 작성 유도**: 회원의 최소 닉네임 입력 보장 및 필수 정보 부족 시 프로필 작성 유도.
- **[0] FR-COM-07. 첨부파일 업로드**: 객체 스토리지 기반 포트폴리오/작업물 업로드.
- **[0] FR-COM-08. 공통 상태 처리**: 로딩 UI, 빈 상태 메시지 제공, 404/오류 페이지 처리.

## 4. 페이지별 요구사항

### 4-1. 홈 `[탐색/검색 중심]`
- **[✴️] FR-HOME-01. 프로젝트 목록 조회**: 카드/리스트형 전환, 제목/인원/날짜/기술스택 등 표시.
- **[✴️] FR-HOME-02. 페이징**: 목록 탐색.
- **[✴️] FR-HOME-03. 프로젝트 검색**: 키워드 (제목, 기술, 포지션) 검색.
- **[✴️] FR-HOME-04. 필터 기능**: 포지션, 스택, 온/오프라인, 유형별 필터.
- **[✴️] FR-HOME-05. 보기 방식 전환**: 카드 형식 / 리스트 형식.
- **[✴️] FR-HOME-06. 관심 등록**: 찜 기능.
- **[✴️] FR-HOME-07. AI 추천 프로젝트 조회**: 프로필 기반 추천 및 사유 표시.
- **[0] FR-HOME-08. 이동 기능**: 로그인, 마이페이지, 등록 페이지 이동.

### 4-2. 회원가입 / 로그인 `[인증 프로세스]`
- **[0] FR-AUTH-01. 소셜 로그인**: 구글 로그인 연동.
- **[0] FR-AUTH-02. 회원가입 처리**: 최초 로그인 시 가입 절차 진행.
- **[0] FR-AUTH-03. 로그인 후 이동**: 이전 페이지 혹은 홈 복귀.
- **[0] FR-AUTH-04. 최초 회원 프로필 이동**: 로그인 직후 필수 프로필 입력 유도.

### 4-3. 마이페이지 `[개인 및 활동 관리]`
- **[0] FR-MYP-01 ~ 04**: 프로필 조회/수정/항목관리 및 공개 설정 기능.
- **[✴️] FR-MYP-05. 읽은/관심 프로젝트**: 최근 조회/찜 목록.
- **[✅] FR-MYP-06. 지원 현황 조회**: 지원 결과 상태.
- **[✅] FR-MYP-07. 참여 목록 조회**: 진행/종료된 프로젝트.
- **[✴️] FR-MYP-08. 등록 프로젝트 목록 조회**: 팀장 권한으로 등록한 프로젝트들.
- **[0] FR-MYP-09. 별점 리뷰 확인**: 본인 대상 별점 내역.
- **[0] FR-MYP-10. 저장 결과 처리**: 성공/실패 얼럿.

### 4-4. 프로젝트 상세 `[교차 및 결정 지점]`
- **[✴️] FR-DET-01. 프로젝트 상세 조회**: 상세 정보 제공.
- **[✴️] FR-DET-02. 관심 등록**: 찜 연동.
- **[✅] FR-DET-03. 프로젝트 지원**: 포지션 선택 후 지원.
- **[0] FR-DET-04 ~ 05**: 별점 및 첨부파일 정보 조회 기능.

### 4-5. 프로젝트 지원서 `[지원 플로우]`
- **[✴️] FR-APP-01 ~ 02**: 지원서 작성 창 렌더링 (프로필 연동 + 지원 메시지).
- **[✅] FR-APP-03 ~ 04**: 제출 및 결과 처리 (성공/실패 분기).

### 4-6. 프로젝트 등록/수정/삭제 `[팀장 권한]`
- **[✴️] FR-REG-01 ~ 04**: 프로젝트 생성, 폼 작성(제목, 인원, 모집 상태 관리 필요) 및 수정/삭제.
- **[✅] FR-REG-05. 모집 상태 관리**: 모집 여건(모집 중, 완료 등) 제어.
- **[0] FR-REG-06 ~ 07**: 파일 업로드 제어 및 수정 불가 항목(조회수, 날짜) 통제.

### 4-7. 프로젝트 관리 `[지원자 및 현황 관리]`
- **[✅] FR-MNG-01 ~ 03**: 지원자 목록, 정보 조회, 승인/보류/거절 제어.
- **[✅] FR-MNG-04. 모집 현황 조회**: 현재 인원 충족 상태 확인.
- **[0] FR-MNG-05 ~ 06**: 팀 내 파일 관리, 종료 후 리뷰 확인.

### 4-8, 4-9, 4-10. 부가 및 확장 기능
- **리뷰 (FR-REV)**: 종료 시 텍스트 제한, 별점 평가 중심.
- **첨부파일 (FR-FILE)**: 객체 스토리지 연동 업/다운로드 상태 안내.
- **AI 추천 (FR-AI)**: 스택/역할 기반 추천, 로딩/오류 처리.

## 5. 비기능 요구사항
- **NFR-01. 사용성**: 직관적인 접근.
- **NFR-02. 응답성**: 조회, 상세, 검색의 빠른 처리 동작.
- **NFR-03. 보안**: 회원 전용 기능의 적절한 차단 및 보호.
- **NFR-04. 확장성**: 향후 채팅, 알림, 상세 리뷰, 공모전 등 확장을 고려한 설계.
