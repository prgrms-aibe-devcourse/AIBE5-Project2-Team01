# 밋볼(MeetBall) 프로젝트 핵심 기능 구현 보고서

본 문서는 밋볼 프로젝트의 MVP 단계에서 구현된 주요 백엔드 API 및 프론트엔드 연동 기능에 대한 명세와 작동 원리를 정리한 제출용 보고서입니다.

---

## 1. 댓글 및 대댓글 시스템 (Comment System)

### 📌 주요 기능
- **계층형 구조**: 댓글에 대한 답글(대댓글)을 달 수 있는 Self-Referencing 구조를 지원합니다.
- **권한별 차별화**: 
    - **팀장(LEADER)**: 닉네임 옆에 전용 [팀장] 뱃지가 표시되어 가독성을 높였습니다.
    - **팀원(MEMBER)**: 댓글 및 답글 작성이 가능합니다.
    - **일반(GUEST)**: 댓글 읽기 및 일반 댓글 작성은 가능하나, **답글(대댓글) 작성은 제한**됩니다. (UI 및 백엔드 이중 차단)

### ⚙️ 기술 사양
- **API**: `POST /api/projects/{id}/comments`, `GET /api/projects/{id}/comments` (페이징 지원)
- **특징**: JPA의 자기 참조(Parent-Child) 매핑을 활용하여 무한 계층 확장이 가능하도록 설계되었습니다.

---

## 2. 프로젝트 별점 리뷰 (Rating & Review)

### 📌 주요 기능
- **실시간 평점 등록**: 사용자가 별(1~5점)을 클릭하여 팀에 대한 평점을 즉시 남길 수 있습니다.
- **동적 통계 리포트**: 
    - 등록된 모든 별점을 합산하여 **평균 평점**을 소수점 첫째 자리까지 계산합니다.
    - 별점별 분포(5점 몇 %, 4점 몇 % 등)를 계산하여 시각적인 막대 그래프로 렌더링합니다.
- **등록 전 미리보기**: 별을 클릭 시 "X점을 등록하시겠습니까?"라는 컨펌 메시지를 노출하여 사용자 실수를 방지합니다.

### ⚙️ 기술 사양
- **API**: `POST /api/projects/{id}/reviews`, `GET /api/projects/{id}/reviews/summary`
- **특징**: 등록 즉시 프론트엔드에서 Fetch API를 통해 통계 데이터를 재호출하여 화면을 갱신(Zero-Refresh)합니다.

---

## 3. 프로젝트 찜 (Bookmark / Wishlist)

### 📌 주요 기능
- **원터치 토글**: 하트 아이콘을 클릭하여 프로젝트를 찜 목록에 추가하거나 취소할 수 있습니다.
- **상태 유지**: 페이지를 새로고침하더라도 해당 유저가 해당 프로젝트를 찜했는지 여부를 DB에서 판단하여 하트 색상(빨간색/회색)을 유지합니다.
- **실시간 카운트**: 전체 사용자가 누른 찜 개수를 합산하여 숫자로 표시합니다.

### ⚙️ 기술 사양
- **API**: `POST /api/projects/{id}/bookmarks` (Toggle 방식)
- **특징**: 데이터베이스의 Unique Constraints를 활용하여 중복 찜 등록을 원천 차단했습니다.

---

## 4. 첨부 파일 관리 (File Attachment/Storage)

### 📌 주요 기능
- **파일 업로드**: 프로젝트 결과물이나 기획서 등 실제 물리 파일을 서버에 업로드할 수 있습니다. (최대 50MB 지원)
- **파일 다운로드**: 업로드된 파일을 클릭 시 브라우저에서 즉시 다운로드 가능하며, 한글 파일명 깨짐 방지 처리가 적용되었습니다.
- **메타데이터 관리**: 실제 파일 이름, 서버 저장 경로, 파일 크기 등을 DB에 기록하여 목록을 관리합니다.

### ⚙️ 기술 사양
- **API**: `POST /api/projects/{id}/attachments`, `GET /api/projects/{id}/attachments/{fileId}/download`
- **특징**: `UUID`를 활용한 파일명 암호화 저장 방식을 채택하여 파일명 충돌을 방지하고 보안성을 높였습니다.
