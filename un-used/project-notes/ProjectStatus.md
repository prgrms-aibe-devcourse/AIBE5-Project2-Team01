# 📝 MeetBall 개발 진행 현황 정리 (2026-04-17)

여태까지 진행한 마이페이지 및 리뷰 시스템 구현 내역을 한눈에 보기 쉽게 정리해 드립니다.

---

## 1. 마이페이지 (MyPage) 통합 및 UI 개선
마이페이지에서 사용자의 활동 내역을 한눈에 확인할 수 있도록 백엔드 API와 프론트엔드 UI를 연결했습니다.

### ✅ 구현된 탭 목록
*   **프로필 정보**: 본인 여부에 따른 수정 버튼 노출 및 이메일 마스킹 처리
*   **참여 프로젝트**: 현재 참여 중인 프로젝트 목록 조회 (종료된 프로젝트는 '리뷰 가능' 표시)
*   **지원 프로젝트**: 지원한 프로젝트와 진행 상태(대기/승인/거절) 확인
*   **읽은 프로젝트**: 상세 페이지 방문 시 자동 저장된 최근 조회 목록
*   **관심 프로젝트**: '찜'한 프로젝트 목록 및 상세 이동 기능
*   **받은 리뷰**: 타인에게 받은 피어 리뷰 목록 및 실시간 밋볼 지수(매너 온도) 반영

---

## 2. 프로젝트 읽기 기록 (Project Read History)
상세 페이지 방문 시 자동으로 사용자의 '최근 본 목록'에 저장되는 기능입니다.

*   **동작 방식**: 프로젝트 상세 페이지 접속 시 백엔드로 읽기 기록 API(`POST /api/projects/{id}/read`) 호출
*   **로직**: 이미 읽은 프로젝트라면 기존 기록의 시간만 업데이트하여 최신순 정렬 유지
*   **연동**: 마이페이지 '읽은 프로젝트' 탭에서 실시간 확인 가능

---

## 3. 리뷰 시스템 (Review & Peer Review)
프로젝트 종료 후 서비스의 신뢰도를 높이기 위한 핵심 기능입니다.

### 🌟 주요 기능
*   **프로젝트 리뷰**: 대상을 지정하지 않고 프로젝트 전체에 대한 별점과 후기 남기기
*   **팀원 피어 리뷰**: 함께 작업한 팀원들을 지목하여 개별 평가 점수 남기기
*   **밋볼 지수(온도) 연동**: 받은 피어 리뷰 점수에 따라 36.5도에서 시작하는 온도가 오르내림

### 🛡️ 보안 및 검증
*   **참여 멤버 전용**: 실제로 해당 프로젝트에 참여한 멤버만 리뷰 작성 가능
*   **중복 작성 방지**: 한 프로젝트에서 특정인(또는 프로젝트)에 대해 리뷰를 한 번만 남길 수 있게 제한
*   **자기 리뷰 금지**: 본인 스스로에게 점수를 주는 부정행위 원천 차단

---

## 4. API 엔드포인트 현황 (개발용)

| 기능 | HTTP 메서드 | 엔드포인트 | 비고 |
|:--- |:---:|:--- |:--- |
| **읽기 기록 저장** | `POST` | `/api/projects/{projectId}/read` | 상세 페이지 진입 시 자동 호출 |
| **팀원 목록 조회** | `GET` | `/api/projects/{id}/reviews/teammates` | 리뷰 작성을 위한 팀원 리스트 |
| **리뷰 등록** | `POST` | `/api/projects/{id}/reviews` | 프로젝트 및 피어 리뷰 통합 |
| **마이페이지 통합** | `GET` | `/api/mypage/{section}` | 프로필, 지원, 참여, 찜 등 조회 |
